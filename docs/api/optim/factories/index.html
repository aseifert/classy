class Factory()

##

Factory interface that allows for simple instantiation of optimizers and schedulers for PyTorch Lightning. This class is essentially a work-around for lazy instantiation: * all params but for the module to be optimized are received in init * the actual instantiation of optimizers and schedulers takes place in the call method, where the module to be optimized is provided call will be invoked in the configure_optimizers hooks of LighiningModule-s and its return object directly returned. As such, the return type of call can be any of those allowed by configure_optimizers, namely: * Single optimizer * List or Tuple - List of optimizers * Two lists - The first list has multiple optimizers, the second a list of LR schedulers (or lr_dict) * Dictionary, with an â€˜optimizerâ€™ key, and (optionally) a â€˜lr_schedulerâ€™ key whose value is a single LR scheduler or lr_dict * Tuple of dictionaries as described, with an optional â€˜frequencyâ€™ key * None - Fit will run without any optimizer
